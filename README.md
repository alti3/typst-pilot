**Typst Pilot** is a next-generation, browser-based editor for [Typst](https://typst.app/), the modern successor to LaTeX. 

Inspired by the ease of use of *Overleaf* and the intelligence of *OpenAI Prism*, Typst Pilot combines a robust three-pane editing environment with a local-first philosophy. It runs entirely on your device using WebAssembly (WASM), ensuring your data never leaves your browser unless you want it to.

> **Why Typst?** Typst is a new markup-based typesetting system that is as powerful as LaTeX but much easier to learn, faster to compile, and provides instant feedback.

## 🎯 Core Philosophy

Scientific and professional writing tools have stagnated. Users often have to choose between:
1.  **Online SaaS tools (like Overleaf):** Great UX, but requires constant internet, introduces latency, and raises data privacy concerns for sensitive research.
2.  **Local CLI workflows:** Private and fast, but lacks integrated visualization, AI assistance, and ease of setup.

**Typst Pilot bridges this gap.** 

It offers the UX of a cloud SaaS—complete with file sidebars, split-pane editing, and integrated chat—but operates entirely locally. By leveraging the Typst compiler compiled to WebAssembly, Typst Pilot brings the compile engine to *your* data, not the other way around.

## ✨ Key Features

### 🚀 Instant Local Compilation
Forget the "Compiling..." loading spinner. Typst Pilot uses `@myriaddreamin/typst.ts` to run the compiler inside your browser's main thread (or worker). Changes are reflected in the PDF preview panel in milliseconds as you type.

### 🤖 Intelligent AI Copilot
Integrated directly into the editor workflow is a context-aware AI assistant.
- **Contextual Awareness:** The AI knows the content of your active document.
- **Code Generation:** Ask it to "Create a table for 3 variables" or "Write a math formula for Standard Deviation," and it generates valid Typst syntax.
- **One-Click Apply:** Apply code blocks generated by the AI directly into your document with a single click.

### 📂 File Management
- **Project Structure:** Create, rename, and delete `.typ` files in a sidebar explorer.
- **Multi-File Support:** Split your document into chapters and imports easily.
- **Persistence:** All files are automatically saved to your browser's `localStorage` (via IndexedDB wrappers), ensuring you don't lose work if you refresh the tab.

### ⚡ Modern Editing Experience
- **Monaco Editor:** Built on the same engine as VS Code, featuring syntax highlighting, line numbers, and minimap.
- **Dark Mode Native:** A carefully crafted dark theme using the `zinc` color palette for reduced eye strain during late-night writing sessions.
- **Real-time PDF Preview:** High-fidelity vector rendering of your document with zoom and pan controls.

## 🏗 Architecture & Tech Stack

Typst Pilot is a Single Page Application (SPA) built with performance and modularity in mind.

### The Stack
- **Frontend Framework:** [React 18](https://react.dev/) with [Vite](https://vitejs.dev/) for lightning-fast HMR.
- **Language:** TypeScript for type safety and developer ergonomics.
- **Styling:** [Tailwind CSS](https://tailwindcss.com/) combined with [shadcn/ui](https://ui.shadcn.com/) for a accessible, consistent component design system.
- **Editor Engine:** `monaco-editor` via `@monaco-editor/react`.
- **Compiler:** `typst-ts` (WASM bindings for Typst).
- **State Management:** Custom React Hooks (`useFileStore`, `useAISettings`) leveraging local persistence.
- **Icons:** Lucide React.

### How Compilation Works
1.  The user types in the Monaco editor.
2.  A debounced listener captures the content.
3.  The content is passed to the WASM module initialized in memory.
4.  The WASM module simulates a file system, compiles the Typst code into a PDF artifact.
5.  The resulting blob is rendered into an iframe/canvas in the Preview panel.
6.  *Result:* Near-native performance with zero server latency.

## 🛠 Getting Started

### Prerequisites
- Node.js 18.0 or higher
- Bun (latest)

### Installation

1.  **Clone the repository**
    ```bash
    git clone https://github.com/yourusername/typst-pilot.git
    cd typst-pilot
    ```

2.  **Install dependencies**
    ```bash
    bun install
    ```

3.  **Run the development server**
    ```bash
    bun run dev
    ```

4.  Open `http://localhost:8080` in your browser.

### Building for Production
To create a static build that can be hosted on GitHub Pages, Vercel, or Netlify:

```bash
bun run build
```
The artifacts will be in the `dist` folder.

## 🤖 The AI Copilot

Typst Pilot does not proxy your AI requests through a backend service. We believe in **BYOK (Bring Your Own Key)**. This ensures:
1.  **Privacy:** Your prompts go directly from your browser to the AI provider. We never see them.
2.  **Cost:** You pay standard API rates directly to the provider, no markup.
3.  **Choice:** Switch between providers at will.

### Supported Providers
- **OpenAI:** `gpt-5.2`, `gpt-5.2-pro`, `gpt-5.1`, `gpt-5`, `gpt-5-mini`, `gpt-5-nano`, `gpt-4.1`, `gpt-4.1-mini`, `gpt-4.1-nano`, `gpt-4o`, `gpt-4o-mini`, `o3`, `o3-mini`, `o4-mini`
- **Anthropic:** `claude-sonnet-4-5-20250929`, `claude-haiku-4-5-20251001`, `claude-opus-4-5-20251101`
- **Google:** `gemini-3-pro-preview`, `gemini-3-flash-preview`, `gemini-2.5-pro`, `gemini-2.5-flash`, `gemini-2.5-flash-lite`, `gemini-2.0-flash`, `gemini-2.0-flash-lite`
- **OpenRouter:** `openrouter/auto`, `openai/gpt-5.2`, `openai/gpt-5.2-pro`, `openai/gpt-5.1`, `openai/gpt-5`, `openai/gpt-5-mini`, `openai/gpt-5-nano`, `openai/gpt-4.1`, `openai/gpt-4.1-mini`, `openai/gpt-4.1-nano`, `openai/gpt-4o`, `openai/gpt-4o-mini`, `openai/o3`, `openai/o3-mini`, `openai/o4-mini`, `anthropic/claude-opus-4.5`, `anthropic/claude-sonnet-4.5`, `anthropic/claude-haiku-4.5`, `google/gemini-3-pro-preview`, `google/gemini-3-flash-preview`, `google/gemini-2.5-pro`, `google/gemini-2.5-flash`, `google/gemini-2.5-flash-lite`
- **Custom Endpoint:** Connect to a local LLM running via Ollama or LM Studio.

### Configuration
1.  Click the **Settings** (gear icon) in the top right.
2.  Select your provider.
3.  Paste your API Key.
4.  Click "Save". The key is encrypted and stored in your browser's LocalStorage.

## 🔒 Privacy & Security

Typst Pilot is designed with a "Local-Only" architecture.

1.  **No Document Uploads:** Your Typst files are stored in your browser's storage. They are never sent to a cloud server.
2.  **Client-Side Compilation:** The compilation logic runs in WASM on your CPU.
3.  **Direct API Calls:** AI chats are sent directly to the API provider you configure.
4.  **Data Export:** You can download your `.typ` source files and the compiled `.pdf` at any time.

*Note: If you clear your browser cache/data, you will lose your documents. We recommend regularly downloading your source files or awaiting our upcoming file-system sync feature.*

## 🎮 Usage Guide

### The Layout
- **Left Panel (File Explorer):** Manage your project files. 
    - Click `+` to add a file. 
    - Right-click (or hover) to rename/delete.
- **Center Panel (Editor):** Write your code here.
    - `Cmd/Ctrl + S` triggers a manual save/compile (though auto-compile is active).
    - `Cmd/Ctrl + B` toggles the sidebar.
- **Bottom Center (AI Chat):**
    - Type naturally: *"How do I make a 3-column layout?"*
    - The AI will return code snippets. Click "Apply" to insert them into your doc.
- **Right Panel (Preview):**
    - Automatically updates.
    - Use the toolbar to download the PDF.

### Writing Typst
Typst is intuitive. Here is a quick cheat sheet:

```typst
= Heading 1
== Heading 2

- Bullet point
- Another point

$ a^2 + b^2 = c^2 $  // Math mode
```
